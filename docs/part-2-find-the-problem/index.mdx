# Part 2 — Find and Validate the Problem

Every great product solves a real problem. But most developers start with a solution and go looking for a problem to fit it.

Part 2 flips that. You'll learn to identify yourself as a builder, read markets like an entrepreneur, think problem-first instead of solution-first, and validate your assumptions before writing code.
