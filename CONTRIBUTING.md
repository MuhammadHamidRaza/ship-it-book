# Contributing to Ship It

Thank you for considering contributing to this book. All contributions — big or small, code or content — are welcome.

## Ways to Contribute

### 📝 Content Improvements
- Fix typos, grammar, or unclear phrasing
- Add examples that illustrate concepts better
- Suggest new chapters or sections
- Improve existing chapters with better explanations
- Translate chapters into other languages

### 💻 Code Improvements
- Fix bugs in the Docusaurus site
- Improve the site design, accessibility, or performance
- Add features like search, dark mode refinements, or analytics
- Improve build and deployment configuration

### 🔍 Feedback
- Open an issue about something that's unclear or incorrect
- Suggest topics you'd like covered
- Report broken links or rendering issues

## Getting Started

1. **Fork the repository**
2. **Create a branch**: `git checkout -b my-improvement`
3. **Make your changes**
4. **Test**: If changing code, run `npm run build` to verify
5. **Commit**: Use clear commit messages
6. **Push**: `git push origin my-improvement`
7. **Open a Pull Request**

## Style Guide

### Writing
- Write like a senior developer talking to a junior friend — direct and honest
- Short paragraphs (max 4 lines)
- Active voice only
- No corporate jargon or MBA-speak
- Every chapter needs: an opening hook, clear sections, real examples, "The Hard Truth" section, a "Your Turn" exercise, and "What You Learned" summary

### Code
- Follow the existing code style in the project
- Use TypeScript for any new code
- Test your changes with `npm run build`

## Questions?

Open a discussion or issue. We're happy to help.

---

All contributors are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md). Be respectful, constructive, and assume good faith.
